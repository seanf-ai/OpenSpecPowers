# SEO Guide (GitHub + npm)

This project is optimized for discovery in GitHub and npm search.

## Core Metadata

- Repository name: `OpenSpecPowers`
- npm package name: `@seanf-ai/openspecpowers`
- Description: concise, keyword-rich, and user-focused
- Topics (GitHub): `ai`, `cli`, `developer-tools`, `spec-driven-development`, `llm`, `workflow`, `typescript`

## README Optimization

Keep these near the top:

- clear product statement
- install command
- quick demo workflow
- badges for CI, npm version, and stars
- links to docs and issue tracker

## Content Strategy

- Use release notes consistently (`CHANGELOG.md`)
- Add practical examples and real-world use-cases
- Keep command docs updated and searchable
- Include terms users search for:
  - spec-driven development
  - AI coding workflow
  - agentic CLI
  - LLM engineering workflow

## Distribution Strategy

- Publish npm package under your own scope
- Keep versioning clean and semantic
- Use GitHub Releases for each tagged version
- Include installer script and copy-paste commands
